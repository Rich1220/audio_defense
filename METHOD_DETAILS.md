# Method Details: Hidden-State Linear Router Defense

This document explains the method, metrics, training flow, and defense simulation in detail.

## 1. Problem Setup

We have a target audio-language model `M`.

For each benchmark example `i`, we have:

```text
x_i = input audio/text prompt
r_i = response generated by M
y_i = safety label from offline judge
```

The label convention is:

```text
y_i = 1  unsafe
y_i = 0  safe
```

The goal is to train a router:

```text
g(features_i) -> risk_score_i in [0, 1]
```

A deployment policy would be:

```text
if risk_score_i >= threshold:
    route to safer path
else:
    pass original response path
```

In our offline simulation, the "safer path" is modeled as converting routed examples to safe outcomes.

## 2. Feature Extraction

For each example, run the target model and collect hidden states.

For a model with `L` hidden layers and hidden size `D`, each layer has a hidden-state matrix:

```text
H_i,l in R^(T_i x D)
```

Where:

- `i` = example index.
- `l` = layer index.
- `T_i` = sequence length for example `i`.
- `D` = hidden dimension.

We convert each variable-length hidden-state matrix into fixed-length feature vectors using pooling.

### 2.1 Pooling Methods

#### `last`

Use the hidden vector at the last valid context token:

```text
z_i,l,last = H_i,l[t_last]
```

This often captures the model's final pre-generation state.

#### `mean_context`

Mean-pool over all valid context tokens:

```text
z_i,l,mean_context = mean_{t in C_i} H_i,l[t]
```

Where `C_i` is the set of valid non-padding context positions.

#### `audio_tokens`

Mean-pool over positions corresponding to inserted audio tokens:

```text
z_i,l,audio_tokens = mean_{t in A_i} H_i,l[t]
```

Where `A_i` is the set of audio-token positions.

If the model does not expose explicit audio-token positions, the extractor may approximate or fall back to context positions, but this should be documented.

### 2.2 Optional Prefix / Logit Scores

Some extractors also save logit-derived scores:

```text
scores_i = [
    P(first token in refusal-prefix set),
    P(first token in transcription-prefix set),
    P(first token in answer-prefix set)
]
```

Example prefix sets:

```text
refusal:       "I'm sorry", "I cannot", "I can't", "I'm unable", ...
transcription: "The request is", "The audio says", ...
answer:        "Here is", "Sure", "Certainly", "First", ...
```

These are optional auxiliary features. The main defense method is hidden-state based.

## 3. Data Format

The training scripts expect:

```text
hidden_features.npz
hidden_meta.jsonl
```

### 3.1 NPZ Arrays

Required arrays:

```text
hidden_last          [N, L, D]
hidden_mean_context  [N, L, D]
hidden_audio_tokens  [N, L, D]
scores               [N, K]
labels               [N]
ids                  [N]
score_names          [K]
pool_names           [3]
```

Where `labels` must be integer 0/1.

### 3.2 Metadata Rows

The JSONL metadata should have one row per example:

```json
{
  "id": "example-id",
  "category": "Physical harm",
  "source": "PAIR",
  "attack_type": "black_box",
  "prompt": "original prompt",
  "local_audio": "path/to/audio.wav",
  "llamaguard_label": "unsafe",
  "extract_error": null
}
```

Rows with non-null `extract_error` are excluded from valid training/evaluation indices.

## 4. Linear Probe Training

For one selected layer/pooling feature:

```text
X = [z_i] in R^(N x D)
y = [y_i] in {0,1}^N
```

We train a logistic regression classifier:

```text
p_i = sigmoid(w^T normalize(z_i) + b)
```

The implementation uses:

```text
sklearn.linear_model.LogisticRegression(
    solver="liblinear",
    class_weight="balanced",
    max_iter=300 or 500
)
```

### 4.1 Normalization

Training features are standardized:

```text
mu_j = mean(X_train[:, j])
sigma_j = std(X_train[:, j])
X_norm[:, j] = (X[:, j] - mu_j) / max(sigma_j, epsilon)
```

The same training-set `mu` and `sigma` are used for validation/test.

### 4.2 Class Imbalance

Unsafe examples are usually a minority. `class_weight="balanced"` compensates for imbalance during logistic-regression fitting.

This does not change reported metrics. It only affects training.

## 5. Splits

### 5.1 Random Split

The default random split is stratified:

```text
70% train
30% test
```

The train split is further split into:

```text
80% fit
20% validation
```

The validation set is used for threshold selection and layer selection in auto-layer experiments.

### 5.2 Held-Out Source Split

For a source-heldout split:

```text
test = all examples from source S
train = all examples not from source S
```

A split is only kept if both train and test contain safe and unsafe examples.

Purpose:

```text
Measure attack-source generalization.
```

### 5.3 Held-Out Category Split

For a category-heldout split:

```text
test = all examples from category C
train = all examples not from category C
```

Purpose:

```text
Measure harmful-topic generalization.
```

Category-heldout results can be noisy when category counts are small.

## 6. Single-Layer Probe Sweep

The single-layer probe script sweeps:

```text
pooling in {last, mean_context, audio_tokens}
layer in all layers
```

For each candidate `(pooling, layer)`, train a linear probe and report AUROC/AUPRC on test.

The best random-split probe is selected by AUROC in the summary.

For model comparison, report:

```text
best pooling
best layer
AUROC
AUPRC
number of unsafe positives in test
```

## 7. Router Defense Simulation

The router defense simulation treats high-risk examples as routed to a safe path.

For each test example:

```text
route_i = 1 if p_i >= threshold else 0
```

If `route_i = 1`, the simulation assumes the unsafe response is prevented.

### 7.1 Confusion Counts

Using:

```text
y_i = 1 means unsafe
route_i = 1 means routed
```

Definitions:

```text
TP = count(route_i = 1 and y_i = 1)  caught unsafe
FP = count(route_i = 1 and y_i = 0)  safe false-routed
TN = count(route_i = 0 and y_i = 0)  safe passed
FN = count(route_i = 0 and y_i = 1)  unsafe missed
N  = TP + FP + TN + FN
```

### 7.2 Defense Metrics

Unsafe rate before routing:

```text
unsafe_before = (TP + FN) / N
```

Unsafe rate after routing:

```text
unsafe_after = FN / N
```

Relative unsafe reduction:

```text
relative_reduction = (unsafe_before - unsafe_after) / unsafe_before
                   = TP / (TP + FN)
```

This is equivalent to caught unsafe recall.

Route rate:

```text
route_rate = (TP + FP) / N
```

Safe false-route rate:

```text
safe_false_route = FP / (FP + TN)
```

Precision:

```text
precision = TP / (TP + FP)
```

Recall:

```text
recall = TP / (TP + FN)
```

F1:

```text
F1 = 2 * precision * recall / (precision + recall)
```

### 7.3 Threshold Selection

Thresholds are selected on validation data, not test data.

The default grid is:

```text
0.05, 0.10, ..., 0.95
```

Default objective:

```text
f1
```

The objective chooses the threshold with best validation F1, with tie-breakers depending on script.

Other supported objectives:

```text
high_recall
low_route
```

Use `high_recall` when safety coverage is more important than utility.

Use `low_route` when minimizing false routes / user friction is more important.

## 8. AUROC and AUPRC

### 8.1 AUROC

AUROC measures how well risk scores rank unsafe examples above safe examples.

Equivalent interpretation:

```text
AUROC = P(score_unsafe > score_safe)
```

AUROC is threshold-independent.

### 8.2 AUPRC

AUPRC measures precision-recall tradeoff for the unsafe class.

It is especially important when unsafe examples are rare.

When unsafe prevalence is low, AUPRC is often more informative than AUROC.

## 9. Auto-Layer Router

The auto-layer router avoids hard-coding layers from one model into another.

For each split:

1. Split train into fit/validation.
2. Sweep all `(pooling, layer)` candidates on validation.
3. Select representations by one of two modes:
   - `depth_regions`
   - `topk`
4. Train one logistic probe per selected representation.
5. Average predicted risk scores.
6. Select threshold on validation.
7. Evaluate blindly on test.

### 9.1 Depth-Region Selection

Layers are divided into four regions:

```text
shallow: 0% - 25%
middle:  25% - 60%
deep:    60% - 90%
final:   90% - 100%
```

The best validation layer/pooling is selected from each region.

This avoids choosing four nearly identical adjacent layers.

### 9.2 Top-K Selection

`topk` simply chooses the best K candidates by validation AUROC or AUPRC.

This may select adjacent layers, which can be useful but less diverse.

### 9.3 Metric for Layer Selection

Use:

```text
--metric auroc
```

for stable ranking.

Use:

```text
--metric auprc
```

when unsafe examples are very rare and you care more about the positive class.

## 10. What Counts as a Defense Claim

A responsible defense claim should include:

```text
unsafe_before
unsafe_after
relative_reduction
route_rate
safe_false_route
threshold objective
random split result
source-heldout result
category-heldout result
```

Do not claim the defense is solved from random split alone.

Recommended wording:

> Hidden-state routing provides a useful internal risk signal and can reduce unsafe rate in offline simulation, but hidden-only routing is not sufficient as a complete safety system.

Avoid claiming:

> The router prevents all jailbreaks.

or:

> The defense is production-ready.

## 11. How to Extend to Another Benchmark

The benchmark only needs to provide rows with:

```text
id
input audio/text path
category/source/attack_type if available
target model response
safety judge label
```

If your benchmark has no `source`, use:

```text
source = benchmark_name
```

If your benchmark has no `category`, use:

```text
category = unknown
```

Random split will still work. Held-out source/category splits require meaningful group fields.

## 12. Common Failure Modes

### 12.1 Layer Choices Do Not Transfer Across Models

Do not hard-code Qwen2-Audio layers for Ultravox or another model.

Use the auto-layer router to select model-specific layers.

### 12.2 LlamaGuard Labels Are Imperfect

LlamaGuard unsafe may include:

- direct harmful compliance
- unsafe instructions with disclaimers
- risky content embedded in fictional framing

Human audit is recommended for final paper claims.

### 12.3 High Reduction With High False Route

A router can reduce unsafe rate by routing too many safe examples.

Always report:

```text
route_rate
safe_false_route
```

### 12.4 Small Held-Out Groups

Category-heldout splits with few unsafe examples can look very strong or very weak by chance.

Always include:

```text
test positives / test rows
```
